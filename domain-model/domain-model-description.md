## Domain-model-v0.1 
**Σύντομη περιγραφή κλάσεων**

1. Χρήστης (User): Οντότητα που αντιπροσωπεύει κάθε άτομο που χρησιμοποιεί την εφαρμογή
2. Τοποθεσία (Location): Η τοποθεσία μπορεί να αντιπροσωπεύει την θέση ενός περιστατικού, όπου περιλαμβάνει συντεταγμένες και μπορεί να είναι αυτόματη ή χειροκίνητη. Ακόμη, μπορεί να αντιπροσωπεύει και την γεωγραφική θέση του χρήστη για να τον συνδέει στο κατάλληλο φόρουμ.
3. Ειδοποίηση (Notification): Στέλνεται αυτόματη ειδοποίηση σε κοντινούς χρήστες που βρίσκονται εντος μιας προκαθορισμένης ακτίνας με την ακριβή τοποθεία και βασικές πληροφορίες για το περιστατικό.
4. Χάρτης (Maps): Εμφανίζει όλες τις επιβεβαιωμένες τοποθεσίες περιστατικών (ανανεώνεται δυναμικά)
5. Φόρουμ (Forum): Οι χρήστες συνδέονται με βάση την τοποθεσία τους και εκεί μπορούν να κάνουν αναρτήσεις και συζητήσεις για προβληματισμούς τους. Ακόμα, ο χρήστης μπορεί να δημιουργήσει νέο φόρουμ σε περίπτωση που χρειάζεται.
6. Ανάρτηση (Forum_Post): Μπορεί να κάνει ο χρήστης σε συγκεκριμένο φόρουμ.
7. Δεδομένα Καιρόυ (weatherdata) : οντότητα που αποθηκεύει τα δεδομένα καιρού που λαμβάνει το σύστημα από τη ΕΜΥ. Περιλαμβάνει πληροφορίες για τα καιρικά φαινόμενα των επόμενων ημερών
8. Ειδοποίηση εντόνων καιρικών φαινομένων (weatheralert) : οντότητα που διαχειρίζεται τις ειδοποιήσεις που στέλνονται από το σύστημα στον χρήστη για την προειδοποίηση έντονων καιρικών φαινομένων.
9.Weatherservice: Οντότητα που αντιπροσωπεύει την εξωτερική υπηρεσία καιρού απο την οποία λαμβάνονται τα δεδομένα.
10. . Δωρεά (donation): οντότητα που καταγράφει την δωρεά που κάνει ένας χρήστης . Περιλαμβάνει το ποσό , την ημερομηνία δωρεάς και το αναγνωριστικό της δωρεας.
11. Κοινωνικές Δράση(socialactivity): οντότητα που αναφέρεται στις διάφορες κοινωνικές δράσεις που διοργανώνονται μέσω της εφαρμογής
12. Περιβαλλοντικός Οργανισμός (environmentalorganisation) : οντότητα που περιέχει όλους τους διαθέσιμους περιβαλλοντικούς οργανισμούς
13. Σύστημα πληρωμών (paymentsystem) : οντότητα που διαχειρίζεται τις πληρωμές για δωρεές
14. AISystem: οντότητα η οποία αντιπροσωπεύει τη διαδικασία και το αποτέλεσμα της αυτοματοποιημένης ανάλυσης που διεξάγεται από τεχνητή νοημοσύνη για κάθε υποβληθείσα αναφορά κινδύνου.
15. Αναφορά(report): Οντότητα που αντιπροσωπεύει μια αναφορά κινδύνου από έναν χρήστη. Περιλαμβάνει πληροφορίες για το περιστατικό, όπως ημερομηνία, τοποθεσία, τύπος, φωτογραφία
16. Φιλτράρισμα(Filter): Οντότητα που αντιστοιχεί σε κριτήρια επιλογής δεδομένων (περίοδος, περιοχή, είδος συμβάντος) που εφαρμόζονται στα στατιστικά.
17. Chatbot : Οντότητα που αντιπροσωπεύει τη διεπαφή αλληλεπίδρασης μεταξύ του εγγεγραμμένου χρήστη και του συστήματος. Παρέχει πληροφορίες, καθοδηγεί τον χρήστη να πλοηγηθεί μέσα στην εφαρμογή και δέχεται αναφορές προβλημάτων.
18. ReportIncident: Οντότητα που αναπαριστά έναν περιβαλλοντικό κίνδυνο που καταγράφεται από τον χρήστη μέσω του συστήματος. Είναι η βασική οντότητα για την αποθήκευση πληροφοριών κινδύνων, που μπορεί να οδηγήσουν στη δημιουργία κοινωνικών δράσεων.
19. SocialActivityNotifications:Οντότητα που διαχειρίζεται τις ειδοποιήσεις που σχετίζονται με τις κοινωνικές δράσεις. Υποστηρίζει την αποστολή μηνύματος επιβεβαίωσης  και μηνύματος ακύρωσης .
20. CreateSocialActivity: Οντότητα που αναπαριστά τη διαδικασία για τη δημιουργία μιας νέας κοινωνικής δράσης από έναν εγγεγραμμένο χρήστη. Είναι η κεντρική οντότητα για το use case "Δημιουργία Κοινωνικής Δράσης", αποθηκεύοντας τις δράσεις στη βάση δεδομένων.
21. EnrollInSocialActivity: Οντότητα που αναπαριστά την εγγραφή ενός χρήστη σε μια υπάρχουσα κοινωνική δράση. Υποστηρίζει το use case "Εγγραφή σε Κοινωνική Δράση", καταγράφοντας τη συμμετοχή και συνδέοντας τις ειδοποιήσεις.
22. Dangermap: Οντότητα που αναπαριστά τον χάρτη με τους κινδύνους π.χ (πυρκαγια) και σημεία που δείχνουν την ακριβή περιοχή του περιστατικού.
23. Στατιστικά(statisticaldata): Οντότητα που περιέχει δεδομένα που έχουν συλλεχθεί από τις αναφορές και άλλες πηγές σχετικά με περιστατικά ανά κατηγορία (πυρκαγιές, ρύπανση, κ.λπ.).
